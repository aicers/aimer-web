# 그룹 허브

그룹 허브는 **고객 그룹**의 분석 화면 진입점입니다. 고객 그룹은 여러
구성원 고객을 묶어 정기 보안 보고서를 함께 생성하고 읽을 수 있게
합니다. 이 버전에서 그룹 허브는 **보고서만** 제공합니다.

![그룹 허브에 대한 자리표시자 — 위협 스토리나 탐지 이벤트 섹션 없이 보안 보고서 카드 하나만 표시 — 실데이터 재캡처 대기 중(그룹 허브는 보고서가 생성된 고객 그룹이 로드된 스택이 필요하며, 픽스처 스택으로는 재현할 수 없습니다)](../../assets/group-hub.ko.png)

## 허브로 이동

사이드바의 **분석 대상 → 그룹** 하위 섹션에서 허브를 바로 엽니다. 이
섹션은 접근 가능한 모든 그룹을 나열하고 각 그룹을 해당
허브(`/subjects/<id>`)로 연결합니다.
[내비게이션](../navigation.md#분석-대상고객-및-그룹)을 참고하세요.

그룹은 고객 범위 선택기에 **범위 프리셋**으로도 나타나지만, 이는 다른
동작입니다. 프리셋은 교차 고객 범위 필터를 그룹 구성원으로 채우고, 허브
링크는 그룹 자체의 보고서 화면을 엽니다.
[그룹 프리셋](../navigation.md#그룹-프리셋)을 참고하세요.

## 섹션

그룹 허브는 단일 섹션 카드를 렌더링합니다.

- **보안 보고서** — 그룹의 정기 보고서 색인([정기 보안
  보고서](reports.md) 참고). 색인, 캘린더, 상세 페이지 모두 그룹의 통합
  보고서를 읽습니다.

위협 스토리나 탐지 이벤트 카드는 의도적으로 **없습니다**. 그룹은 정기
보고서만 보유하므로, 이 버전의 그룹 허브에는 해당 화면이 포함되지
않습니다.

## 그룹 보고서 읽기

그룹 보고서는 그룹 구성원 전체에 걸쳐 생성되며 단일 고객 보고서와
동일하게 읽히되, 두 가지 그룹 고유 동작이 있습니다.

- **인용 출처는 소유 구성원을 가리킵니다.** 보고서에서 인용된 각 스토리나
  이벤트는 해당 증거가 존재하는 **구성원 고객**의 상세 페이지로
  연결됩니다 — 그룹이 아니라 — 따라서 그룹 보고서에서 곧바로 해당 구성원의
  분석으로 들어갈 수 있습니다.
- **생성 동작은 제공되지 않습니다.** 단일 고객 보고서에 표시되는 분석가
  전용 **재생성** 컨트롤과 온디맨드 언어 작업 진행 표시는 이 버전의 그룹
  보고서에서는 렌더링되지 않습니다. 보고서 색인, 캘린더, 기간 내 탐색,
  이미 생성된 언어 간 전환은 평소와 같이 동작합니다.

보고서 **캘린더**는 그룹 고유의 보존 경계 — `min(그룹 보존, 가장 짧은
구성원 보존)` — 를 사용하므로, 탐색 가능한 범위가 어떤 구성원이 아직
보존하는 범위를 넘어서지 않습니다.

## 접근 제어

그룹 허브와 보고서 화면은 **모든 구성원 `reports:read`** 규칙으로
보호됩니다. 그룹의 **모든** 구성원에 대해 `reports:read` 권한을 보유해야
합니다.

- 한 구성원이라도 `reports:read`가 없는 사용자는 `403`을 받습니다.
- 모든 구성원 고객의 **구성원이 아닌** 사용자는 `404`를 받습니다(존재
  은닉 — 비구성원은 그룹의 존재조차 알 수 없습니다).
- 브릿지 세션은 거부됩니다. 단일 고객 보고서 화면과 마찬가지로 그룹
  화면은 브릿지에서 읽을 수 없습니다(`403`).
