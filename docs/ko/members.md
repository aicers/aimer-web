# 멤버

멤버 페이지에서 매니저는 고객 워크스페이스에 접근할 수 있는
사용자를 조회, 초대, 관리할 수 있습니다. 사이드바에서
**멤버**를 클릭하여 페이지를 엽니다.

**매니저** 역할의 사용자만 멤버 페이지를 볼 수 있고 관리 작업을
수행할 수 있습니다. **사용자** 역할의 사용자에게는 이 페이지가
사이드바에 표시되지 않습니다.

![멤버 페이지](../assets/members-table.png)

## 멤버 테이블

테이블에는 선택한 고객 워크스페이스의 모든 현재 멤버가
나열됩니다. 각 행에는 다음 정보가 표시됩니다:

- **이름** — 멤버의 표시 이름. 본인의 행에는 "(나)" 라벨이
    표시됩니다.
- **이메일** — 멤버의 이메일 주소.
- **역할** — 사용자 또는 매니저.
- **마지막 로그인** — 멤버의 가장 최근 로그인 날짜와 시간,
    또는 로그인한 적이 없으면 "없음".

## 역할

각 멤버에게는 두 가지 역할 중 하나가 할당됩니다:

- **사용자** — 대시보드에 접근하고 이벤트, 분석, 보고서 등
    표준 기능을 사용할 수 있습니다.
- **매니저** — 사용자의 모든 권한에 더해 멤버 초대 및 제거,
    멤버 역할 변경, 고객 설정 구성 권한이 있습니다.

모든 워크스페이스에는 최소 한 명의 매니저가 있어야 합니다.
마지막 남은 매니저는 제거하거나 강등할 수 없습니다. 해당 멤버
이름 옆에 경고 배지가 표시됩니다.

## 새 멤버 초대

1. 멤버 섹션 오른쪽 상단의 **초대** 버튼을 클릭합니다.
2. 나타나는 대화 상자에서 이메일 주소를 입력하고 역할(사용자
    또는 매니저)을 선택합니다.
3. **초대 보내기**를 클릭합니다.

![멤버 초대 대화 상자](../assets/invite-dialog.png)

지정된 주소로 초대 이메일이 발송됩니다. 초대는 7일간
유효합니다.

해당 이메일 주소가 이미 워크스페이스의 멤버인 경우 오류 메시지가
표시됩니다.

## 대기 중인 초대

멤버 테이블 아래에 **대기 중인 초대** 섹션이 표시되며, 발송되었
으나 아직 수락되지 않은 초대를 보여줍니다. 각 행에는 다음 정보가
표시됩니다:

- **이메일** — 초대된 이메일 주소.
- **역할** — 초대에 할당된 역할.
- **만료 일시** — 초대가 만료되는 시점.

![대기 중인 초대](../assets/pending-invitations.png)

초대를 취소하려면 **철회**를 클릭하고 대화 상자에서 확인합니다.
초대 링크는 즉시 무효화됩니다.

## 초대 수락

사용자가 초대 이메일을 받으면 수락 링크를 클릭합니다. 흐름은
다음과 같습니다:

1. 링크를 클릭하면 Clumit Insight 로그인 페이지로 리디렉트됩니다.
2. 로그인 후, Clumit Insight은 로그인 이메일이 초대된 이메일과
    일치하는지, 그리고 Keycloak에서 이메일이 인증되었는지
    확인합니다.
3. 확인이 통과하면 사용자가 할당된 역할로 고객 워크스페이스에
    자동으로 추가됩니다.

확인에 실패하면(이메일 불일치, 이메일 미인증 또는 초대 만료)
사용자는 적절한 오류 메시지가 표시된 접근 거부 페이지로
리디렉트됩니다([인증](authentication.md) 참조).

매니저는 동일한 이메일 주소로 다시 초대할 수 있습니다. 이 경우
새 초대 링크가 생성되고 7일 만료 기간이 초기화됩니다.

## 멤버 역할 변경

멤버 이름 옆의 **역할 변경**을 클릭합니다. 확인 대화 상자에
새 역할이 표시됩니다. **역할 변경**을 클릭하여 확인합니다.
변경 사항은 즉시 적용됩니다.

워크스페이스의 마지막 매니저는 역할을 변경할 수 없습니다.

## 멤버 제거

멤버 이름 옆의 **제거**를 클릭합니다. 확인 대화 상자에서 제거를
확인합니다. **제거**를 클릭하여 확인합니다. 해당 멤버는 즉시
워크스페이스에 대한 접근 권한을 잃습니다.

워크스페이스의 마지막 매니저는 제거할 수 없습니다.
