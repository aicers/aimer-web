# 환경 관리

환경 페이지에서 시스템 관리자가 AICE 환경을 등록, 조회, 편집,
삭제하고, 고객-환경 매핑을 관리하며, 신뢰 레지스트리 키를
관리할 수 있습니다. 관리자 사이드바에서 **환경**을 클릭하여
열 수 있습니다.

이 페이지를 조회하려면 `aice-environments:read` 권한이 있는
시스템 관리자여야 합니다. 환경을 생성, 편집, 삭제하고 고객
매핑을 관리하려면 `aice-environments:write` 권한이 필요합니다.
신뢰 레지스트리 작업에는 `trust-registry:read`와
`trust-registry:write` 권한이 필요합니다.

![환경 페이지](../assets/admin-environments-table.png)

## 환경 테이블

테이블에 시스템의 모든 AICE 환경이 표시됩니다. 각 행에는 다음
정보가 표시됩니다:

- **AICE ID** — 환경의 고유 식별자.
- **이름** — 환경의 표시 이름, 선택적 설명이 아래에 표시됩니다.
- **상태** — 활성, 정지, 비활성 중 하나.
- **고객** — 이 환경에 연결된 고객 수. 숫자를 클릭하면 매핑을
    관리할 수 있습니다.
- **키** — 등록된 신뢰 레지스트리 키 수. 숫자를 클릭하면 키를
    관리할 수 있습니다.
- **작업** — 편집 및 삭제 버튼.

## 환경 생성

1. 오른쪽 상단의 **환경 생성** 버튼을 클릭합니다.
2. 필수 항목을 입력합니다:
    - **AICE ID** — 고유한 영숫자 식별자(하이픈 및 밑줄 허용).
    - **이름** — 환경의 표시 이름.
    - **설명** — 선택 사항.
    - **상태** — 초기 상태(기본값: 활성).
3. 선택적으로 **신뢰 레지스트리 키**를 체크하여 환경과 함께
    키를 등록할 수 있습니다:
    - **발급자** — 토큰 발급자 식별자.
    - **키 ID (kid)** — 키 식별자.
    - **공개 키 (JWK)** — JWK JSON을 붙여넣습니다.
    - **키 설명** — 선택 사항.
4. **환경 생성**을 클릭하여 제출합니다.

![환경 생성 대화상자](../assets/admin-environments-create-dialog.png)

## 환경 편집

1. 작업 열의 **편집** 버튼을 클릭합니다.
2. 이름, 설명 또는 상태를 필요에 따라 수정합니다.
3. **저장**을 클릭하여 변경 사항을 적용합니다.

## 환경 삭제

1. 작업 열의 **삭제** 버튼을 클릭합니다.
2. 모든 고객 매핑과 신뢰 레지스트리 키가 영구적으로
    삭제된다는 경고와 함께 확인 대화상자가 나타납니다.
3. **삭제**를 클릭하여 확인합니다.

![삭제 확인 대화상자](../assets/admin-environments-delete-dialog.png)

삭제하면 환경 레코드, 모든 관련 고객 매핑, 모든 신뢰
레지스트리 키가 제거됩니다. 이 작업은 되돌릴 수 없습니다.

## 고객 매핑 관리

**고객** 열의 숫자를 클릭하면 고객 매핑 패널이 열립니다.

### 고객 연결

1. **고객 연결** 버튼을 클릭합니다.
2. 드롭다운에서 고객을 선택합니다. 이미 이 환경에 연결된
    고객은 표시되지 않습니다.
3. **고객 연결**을 클릭하여 확인합니다.

### 고객 연결 해제

1. 고객 옆의 **삭제** 버튼을 클릭합니다.
2. 대화상자에서 제거를 확인합니다.

## 신뢰 레지스트리 키 관리

**키** 열의 숫자를 클릭하면 신뢰 레지스트리 패널이 열립니다.

### 키 등록

1. **키 등록** 버튼을 클릭합니다.
2. 필수 항목을 입력합니다:
    - **발급자** — 토큰 발급자 식별자.
    - **키 ID (kid)** — 키 식별자.
    - **공개 키 (JWK)** — JWK JSON을 붙여넣습니다.
    - **키 설명** — 선택 사항.
3. **키 등록**을 클릭하여 제출합니다.

### 키 활성화 또는 비활성화

작업 열에서 **비활성화** 또는 **활성화**를 클릭하여 키의
상태를 전환합니다. 비활성화된 키는 토큰 검증에 사용되지
않습니다.

### 키 제거

1. 작업 열의 **제거** 버튼을 클릭합니다.
2. 대화상자에서 제거를 확인합니다.

키 제거는 영구적이며 되돌릴 수 없습니다.

## 감사 추적

환경 관리 작업은 감사 로그에 기록됩니다:

- **environment.created** — 환경이 등록된 경우.
- **environment.updated** — 환경 세부 정보가 변경된 경우.
- **environment.deleted** — 환경이 삭제된 경우.
- **environment.customer_linked** — 고객이 연결된 경우.
- **environment.customer_unlinked** — 고객 연결이 해제된
    경우.
- **trust_registry.key_registered** — 키가 등록된 경우.
- **trust_registry.key_disabled** — 키가 활성화 또는
    비활성화된 경우.
- **trust_registry.key_removed** — 키가 제거된 경우.

[감사 로그](audit-logs.md) 페이지에서 이 항목들을 확인할 수
있습니다.
