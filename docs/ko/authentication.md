# 인증

Aimer Web은 Keycloak을 ID 제공자로 사용합니다. 사용자는 OpenID
Connect(OIDC) 리디렉트 흐름을 통해 로그인하며, 인증이 완료되면
대시보드로 돌아옵니다.

## 로그인

**로그인**을 클릭하거나 보호된 페이지에 접근하면 브라우저가
Keycloak 로그인 화면으로 리디렉트됩니다. 자격 증명을 입력하면
인증이 완료되고 Aimer Web 대시보드로 돌아옵니다.

![로그인 페이지](../assets/sign-in.png)

계정이 여러 고객 워크스페이스에 속해 있는 경우, 접근 가능한 첫
번째 고객이 선택된 상태로 대시보드가 열립니다. 사이드바의 선택기를
사용하여 고객을 전환할 수 있습니다([내비게이션](navigation.md)
참조).

## 로그아웃

헤더 바 오른쪽의 프로필 드롭다운을 클릭하여 사용자 메뉴를 연
다음 **로그아웃**을 클릭합니다. 로그아웃하면 Aimer Web 세션이
철회되고 Keycloak 로그아웃 페이지로 리디렉트되어 싱글
사인온(SSO) 세션이 종료됩니다.

![프로필 드롭다운의 로그아웃](../assets/sign-out-header.png)

## 관리자 인증

시스템 관리자는 다중 인증(MFA)이 필요한 별도의 로그인 흐름을 통해
관리자 대시보드에 접근합니다. 관리자 인증은 5분 이내에 완료해야
합니다. 관리자 세션이 활성화되어 있어도 일반 세션에는 영향을 주지
않습니다.

## Aimer Console에서의 브릿지 진입

Aimer Web은 Aimer Console(aice-web-next)의 **Aimer로 보내기** 동작을
통한 진입도 허용합니다. Console 운영자가 이 버튼을 클릭하면 브라우저는
Aimer Web의 브릿지 엔드포인트로 최상위 멀티파트 POST를 수행하며,
엔드포인트가 요청을 검증한 뒤 위에서 설명한 동일한 Keycloak 로그인
화면으로 이어집니다.

![브릿지 사인인 (Aimer Console 진입)](../assets/sign-in.png)

브릿지 POST는 항상 운영자, 출처 AICE, 고객 범위를 식별하는 단기
컨텍스트 토큰을 포함합니다. 선택적으로 서명된 탐지 이벤트 봉투와
일치하는 이벤트 페이로드를 함께 전송할 수 있습니다. 초기 릴리스에서는
이벤트 페이로드가 작은 UTF-8 JSON 문서이므로 Aimer Console은 이를
바이너리 파일 업로드 대신 일반 텍스트 폼 필드로 전송합니다. 브릿지
엔드포인트는 두 형태를 모두 허용하며, 동일한 크기 상한과 SHA-256
페이로드 해시 검증을 양쪽에 동일하게 적용합니다.

컨텍스트 토큰이 없거나 만료되었거나 거부되면 Aimer Web은 로그인
흐름을 시작하지 않습니다. 브릿지 엔드포인트는 JSON 오류(토큰 누락은
HTTP 400, 거부 또는 만료된 토큰은 HTTP 403)로 응답하며, Aimer Console이
이를 운영자에게 다시 표시합니다. 이 경우 Aimer Console로 돌아가
**Aimer로 보내기**를 다시 실행하여 새로운 컨텍스트 토큰을
발급받으세요.

![브릿지 거부 페이지 — 고객 스코프 불일치](../assets/auth-bridge-deny.ko.png)

## 접근 거부

인증 또는 권한 부여에 실패하면 Aimer Web은 원인을 설명하는
메시지와 함께 접근 거부 페이지를 표시합니다.

![접근 거부 페이지](../assets/deny-page.png)

주요 원인은 다음과 같습니다:

- **계정 비활성** — 관리자가 계정을 일시 중지하거나 비활성화한
    경우입니다.
- **접근 권한 없음** — 계정이 어떤 고객 워크스페이스에도 속해 있지
    않습니다. 매니저에게 초대를 요청하세요.
- **MFA 필요** — 관리자 접근에는 다중 인증이 필요합니다.
    Keycloak에서 TOTP 또는 WebAuthn 장치를 설정하세요.
- **관리자 세션 만료** — 관리자 인증 유효 시간(5분)이 경과했습니다.
    다시 로그인하세요.
- **초대 만료** — 초대 링크가 더 이상 유효하지 않습니다. 매니저에게
    새 초대를 요청하세요.
- **이메일 불일치** — 초대가 전송된 이메일과 다른 이메일 주소로
    로그인했습니다.
- **이메일 미인증** — Keycloak에서 이메일 주소가 인증되지
    않았습니다. 이메일을 인증한 후 다시 시도하세요.

### 브릿지 로그인 거부 사유

AICE에서 브릿지 연결을 통해 돌아왔을 때 발생하는 거부 사유는
별도로 안내됩니다. 거부 페이지에는 사유별로 사용자가 취해야 할
복구 절차가 함께 표시됩니다:

- **브릿지 만료** — 연결 링크가 만료되었거나 이미 사용되었습니다.
    AICE에서 새로운 연결을 시작하세요.
- **고객 범위 불일치** — AICE에서 전달된 고객 범위가 Aimer Web에
    등록된 환경의 고객과 일치하지 않습니다. 운영자에게 실패 시각을
    함께 알려 등록 정보를 정정해 달라고 요청하세요.
- **고객 비활성** — 고객 레코드는 존재하지만 현재 활성 상태가
    아닙니다. 운영자가 고객을 재활성화한 뒤 다시 시도하세요.
- **환경 비활성** — AICE 환경은 등록되어 있지만 현재 활성 상태가
    아닙니다. 운영자가 환경을 재활성화한 뒤 다시 시도하세요.
- **접근 권한 없음** — 로그인은 완료되었지만, AICE가 접근하라고
    요청한 고객에 대한 권한이 계정에 없습니다. 해당 고객의
    멤버여야 한다면 그 고객의 매니저에게 초대를 요청하세요.
    분석가로 지정되어야 한다면 시스템 관리자에게 분석가 배정을
    요청하세요.

![브릿지 사인인 거부 — 브릿지 만료](../assets/auth-bridge-key-expired.ko.png)

개인정보 보호를 위해 거부 페이지에는 특정 고객이나 환경 이름이
표시되지 않습니다. 관련 식별자는 감사 로그에 기록되며, 시스템
관리자만 열람할 수 있습니다([감사 로그](audit-logs.md) 참조).

각 오류 페이지에는 일반 로그인 흐름으로 돌아가는 **로그인으로
돌아가기** 링크가 포함되어 있습니다. 관리자 또는 브릿지 세션
오류의 경우, 해당 흐름은 별도로 다시 시작해야 합니다.
