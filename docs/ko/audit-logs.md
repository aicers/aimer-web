# 감사 로그

감사 로그 뷰어는 Aimer Web에서 기록된 모든 시스템 이벤트를 읽기
전용으로 검색할 수 있는 화면입니다. 시스템 관리자만 관리자
대시보드에서 접근할 수 있습니다.

## 감사 로그 뷰어 접근

사이드바에서 **관리자 > 감사 로그**로 이동합니다. 뷰어는 최근
항목을 자동으로 불러옵니다.

![감사 로그 뷰어](../assets/audit-logs-viewer.png)

## 필터링

페이지 상단의 필터 바를 사용하여 결과를 좁힐 수 있습니다.

- **인증 컨텍스트** — `일반` 또는 `관리자` 컨텍스트로
  필터링합니다.
- **작업** — 특정 작업 이름으로 필터링합니다
  (예: `customer.created`).
- **행위자 ID** — 특정 행위자(계정 UUID)로 필터링합니다.
- **고객 ID** — 특정 고객 UUID로 필터링합니다.
- **AICE ID** — 특정 AICE 환경 식별자로 필터링합니다.
- **시작일 / 종료일** — 날짜 범위로 필터링합니다.

**적용**을 클릭하여 검색을 실행하거나, **초기화**를 클릭하여 모든
필터를 해제합니다.

![감사 로그 필터](../assets/audit-logs-filters.png)

## 항목 상세 보기

행을 클릭하면 추가 세부 정보가 펼쳐집니다:

- **상세 정보** — 이벤트와 연관된 전체 JSON 페이로드.
- **SID** — 이벤트 발생 시점의 세션 ID.
- **고객** — 해당하는 경우 고객 UUID.
- **AICE 환경** — 해당하는 경우 AICE 환경 ID.
- **상관 ID** — 전체 상관 UUID.

![감사 로그 상세](../assets/audit-logs-details.png)

## 상관 ID 그룹핑

Aimer Web에서 처리되는 각 요청에는 상관 ID가 할당됩니다.
관련 이벤트(예: 인증 확인 후 리소스 생성)는 동일한 상관 ID를
공유합니다.

상관 그룹의 모든 이벤트를 보려면 행의 상관 ID 링크를 클릭합니다.
페이지 상단에 활성 상관 필터를 보여주는 배너가 나타납니다.
**상관 필터 해제**를 클릭하면 일반 보기로 돌아갑니다.

## 페이지네이션

뷰어는 한 번에 50개 항목을 불러옵니다. 추가 항목이 있으면
테이블 하단에 **더 보기** 버튼이 나타납니다. 클릭하면 다음
페이지의 결과가 추가됩니다.
